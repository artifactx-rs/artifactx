# Support

Use the right GitHub surface so maintainers can respond efficiently.

## Usage questions and migration planning

Use [GitHub Discussions](https://github.com/artifactx-rs/artifactx/discussions) for:

- planning an apt/yum migration;
- asking whether ArtifactX fits your repository layout;
- comparing import, mirror, Pages, or self-hosted `arx serve`;
- sharing successful client setup snippets.

## Bugs

Use [Issues](https://github.com/artifactx-rs/artifactx/issues/new/choose) for reproducible defects. Include the command, version, source repository shape, and client error.

## Security

Do not open public issues for vulnerabilities. See [`SECURITY.md`](SECURITY.md).
